# live_camera_app

같은 Wi-Fi에 있는 두 모바일 기기끼리 카메라 영상을 실시간으로 주고받는 데모 앱입니다. **클라우드(서버) 없이** 기기끼리 직접 연결합니다.

- 영상: WebRTC P2P로 기기끼리 직접 전송 (`flutter_webrtc`)
- 시그널링: 한 기기가 작은 WebSocket 서버를 열고, 다른 기기가 그 IP로 접속해 offer/answer를 직접 교환 (Firebase 등 외부 설정 불필요)

한 기기는 "송신자", 다른 기기는 "수신자"를 고릅니다. 역할 전환 탭은 인디케이터가 슬라이드하며 살짝 튕기는(easeOutBack) 애니메이션으로 움직입니다.

## 실행 방법

두 기기를 **같은 Wi-Fi**에 연결한 뒤 각각 실행합니다. (카메라는 실제 기기에서만 동작합니다.)

```bash
flutter run
```

1. A 기기: `송신자` 탭을 고르고 `송신 시작`을 누릅니다. 화면에 `내 IP`가 표시됩니다. (예: 192.168.0.12)
2. B 기기: `수신자` 탭을 고르고, A에 표시된 IP를 입력한 뒤 `수신 시작`을 누릅니다.
3. 잠시 후 두 기기에 서로의 카메라 영상이 나타나면 성공입니다.

> 반드시 송신자(A)를 먼저 시작해야 합니다. 서버가 열려 있어야 수신자가 접속할 수 있습니다.

## 동작 원리

- 송신자가 포트 `8888`에 WebSocket 서버를 엽니다.
- 수신자가 `ws://<송신자 IP>:8888`로 접속하면, 송신자가 offer를 보내고 수신자가 answer를 돌려줍니다.
- ICE 후보는 따로 주고받지 않고, 수집이 끝난 뒤 SDP에 모두 담아 한 번에 교환합니다(non-trickle). 같은 네트워크라 빠르게 끝납니다.
- 이후 영상은 WebRTC P2P로 기기끼리 직접 흐릅니다.

## 참고

- 코드는 `lib/main.dart` 한 파일에 모두 들어 있습니다.
- 같은 네트워크에서 연결이 안 되면, 두 기기가 정말 같은 Wi-Fi인지(게스트망 분리 여부 포함)와 IP 입력값을 확인하세요.
